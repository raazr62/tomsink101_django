# Task Management System

## 🎯 Overview

The Task Management System automatically converts AI-generated workout and diet plans into trackable tasks, allowing users to monitor their fitness progress in real-time.

---

## ✨ Features

### Automatic Plan Creation
- 🤖 **AI Integration**: Automatically saves AI-generated plans as tasks
- 📊 **Progress Tracking**: Track completion percentages for workouts and diets
- ✅ **Status Management**: Mark exercises and meals as completed
- 📅 **Daily Progress**: Log and view daily achievements
- 🎯 **Goal Setting**: Set target completion dates

### Comprehensive Tracking
- **Workout Plans**: Track exercises, sets, and reps
- **Diet Plans**: Monitor calories, protein, carbs, and fats
- **Exercise Progress**: Mark individual sets as completed
- **Meal Tracking**: Log meal completion and add notes
- **Dashboard**: Overview of active plans and progress

---

## 🏗️ Architecture

### Models

#### WorkoutPlan
Stores user workout plans generated by AI.

**Fields:**
- `user` - Owner of the plan
- `chat_session` - Link to AI chat session
- `name` - Plan name
- `summary` - User information summary
- `status` - active/completed/paused
- `start_date` - When plan started
- `target_completion_date` - Goal date

**Properties:**
- `progress_percentage` - Overall completion %
- `total_exercises` - Count of exercises
- `completed_exercises` - Count completed

#### Exercise
Individual exercises within a workout plan.

**Fields:**
- `workout_plan` - Parent plan
- `name` - Exercise name
- `sets` - Number of sets
- `reps` - Reps per set (e.g., "10-12")
- `completed_sets` - Sets completed
- `status` - pending/in_progress/completed/skipped
- `order` - Display order

#### DietPlan
Stores user diet plans generated by AI.

**Fields:**
- `user` - Owner of the plan
- `chat_session` - Link to AI chat session
- `name` - Plan name
- `summary` - User information summary
- `status` - active/completed/paused

**Properties:**
- `total_daily_calories` - Sum of all meals
- `total_daily_protein` - Total protein (g)
- `total_daily_carbs` - Total carbs (g)
- `total_daily_fats` - Total fats (g)

#### Meal
Individual meals within a diet plan.

**Fields:**
- `diet_plan` - Parent plan
- `meal_type` - breakfast/lunch/snack/dinner
- `title` - Meal name
- `items` - JSON array of food items
- `calories`, `protein`, `carbs`, `fats` - Nutrients
- `status` - pending/completed/skipped
- `order` - Display order

#### DailyProgress
Daily tracking log for workout and diet completion.

**Fields:**
- `user` - User
- `date` - Progress date
- `workout_plan` - Active workout plan
- `diet_plan` - Active diet plan
- `exercises_completed` - Count
- `meals_completed` - Count
- `notes` - Daily notes

## 🔄 Integration with AI Chat

### Automatic Workflow

```
User → AI Chat → AI Response → Task System
                     ↓
              Workout Plan Data → WorkoutPlan + Exercises
              Diet Plan Data    → DietPlan + Meals
```

### How It Works

1. **User chats with AI** about fitness goals
2. **AI generates** workout and diet plans
3. **System automatically creates**:
   - WorkoutPlan with all exercises
   - DietPlan with all meals
4. **Response includes** task IDs
5. **User can immediately** track progress

### Example Integration

When AI returns workout and diet data:

```python
# AI Response
{
    "workout": [
        {"exercise": "Push-ups", "sets": 3, "reps": "10-12"},
        {"exercise": "Squats", "sets": 3, "reps": "15-20"}
    ],
    "diet": [
        {
            "meal": "Breakfast",
            "title": "Oatmeal Bowl",
            "items": ["Oats", "Berries", "Honey"],
            "nutrients": {"calories": 350, "protein": 10, "carbs": 60, "fats": 8}
        }
    ],
    "summary": "Male, 23 years old, 80kg, beginner..."
}

# Automatically creates:
# - WorkoutPlan with 2 Exercises
# - DietPlan with meal entries
# - Links to chat session
```

## 📡 API Endpoints

### Base URL: `/api/tasks/`

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/dashboard/` | Overview of active plans |
| GET | `/workout-plans/` | List all workout plans |
| GET | `/workout-plans/<id>/` | Get workout plan details |
| PATCH | `/workout-plans/<id>/` | Update workout plan |
| DELETE | `/workout-plans/<id>/` | Delete workout plan |
| PATCH | `/exercises/<id>/` | Update exercise progress |
| GET | `/diet-plans/` | List all diet plans |
| GET | `/diet-plans/<id>/` | Get diet plan details |
| PATCH | `/diet-plans/<id>/` | Update diet plan |
| DELETE | `/diet-plans/<id>/` | Delete diet plan |
| PATCH | `/meals/<id>/` | Update meal status |
| GET | `/progress/` | Get daily progress |
| POST | `/progress/` | Log daily progress |

See [API_DOCUMENTATION.md](./API_DOCUMENTATION.md) for complete details.

## 💻 Usage Examples

### View Dashboard

```python
import requests

headers = {"Authorization": f"Bearer {token}"}

# Get overview
response = requests.get(
    "http://localhost:8000/api/tasks/dashboard/",
    headers=headers
)
dashboard = response.json()

print(f"Active Workout: {dashboard['active_workout_plan']['name']}")
print(f"Progress: {dashboard['active_workout_plan']['progress_percentage']}%")
```

### Update Exercise

```python
# Mark exercise completed
response = requests.patch(
    f"http://localhost:8000/api/tasks/exercises/{exercise_id}/",
    headers=headers,
    json={
        "completed_sets": 3,
        "status": "completed",
        "notes": "Felt great!"
    }
)
```

### Mark Meal Complete

```python
# Complete a meal
response = requests.patch(
    f"http://localhost:8000/api/tasks/meals/{meal_id}/",
    headers=headers,
    json={
        "status": "completed",
        "notes": "Delicious and nutritious!"
    }
)
```

### View Workout Plan

```python
# Get detailed workout plan
response = requests.get(
    f"http://localhost:8000/api/tasks/workout-plans/{plan_id}/",
    headers=headers
)
plan = response.json()

for exercise in plan['exercises']:
    print(f"{exercise['name']}: {exercise['completion_percentage']}%")
```

## 📊 Database Schema

```
WorkoutPlan
├── id (UUID)
├── user_id (FK → User)
├── chat_session_id (FK → ChatSession)
├── name
├── summary
├── status
└── Exercises (1-to-many)
    ├── id (UUID)
    ├── workout_plan_id (FK)
    ├── name
    ├── sets
    ├── reps
    ├── completed_sets
    └── status

DietPlan
├── id (UUID)
├── user_id (FK → User)
├── chat_session_id (FK → ChatSession)
├── name
├── summary
├── status
└── Meals (1-to-many)
    ├── id (UUID)
    ├── diet_plan_id (FK)
    ├── meal_type
    ├── title
    ├── items (JSON)
    ├── calories, protein, carbs, fats
    └── status

DailyProgress
├── id (UUID)
├── user_id (FK → User)
├── date
├── workout_plan_id (FK)
├── diet_plan_id (FK)
├── exercises_completed
└── meals_completed
```
