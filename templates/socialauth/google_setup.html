🔧 Google OAuth Setup Required

Follow these steps to enable Google Sign-In

⚠️ Configuration Missing

Google Client ID is not configured. Please complete the setup steps below to enable Google authentication.

1

Create Google Cloud Project

Go to Google Cloud Console

  • Click "Select a project" → "New Project"
  • Enter a project name (e.g., "My Django App")
  • Click "Create"
2

Enable Google+ API

  • In the Google Cloud Console, go to "APIs & Services" → "Library"
  • Search for "Google+ API"
  • Click on it and press "Enable"
3

Create OAuth 2.0 Credentials

  • Go to "APIs & Services" → "Credentials"
  • Click "Create Credentials" → "OAuth client ID"
  • If prompted, configure the OAuth consent screen first
  • Choose "Web application" as the application type
  • Add authorized JavaScript origins:
http://localhost:8000 http://127.0.0.1:8000
  • Add authorized redirect URIs:
{{ redirect_uri }}
  • Click "Create"
  • Copy your Client ID and Client Secret
4

Configure Environment Variables

Add these to your .env file in your project root:

GOOGLE_CLIENT_ID=your_client_id_here GOOGLE_CLIENT_SECRET=your_client_secret_here GOOGLE_SECRET_KEY=your_secret_key_here GOOGLE_OAUTH_CALLBACK_URL={{ redirect_uri }}

💡 Tip: Create a .env file in your project root if it doesn't exist. Never commit this file to version control!

5

Verify Settings Configuration

Make sure your settings.py has these lines:

from decouple import config GOOGLE_CLIENT_ID = config('GOOGLE_CLIENT_ID') GOOGLE_CLIENT_SECRET = config('GOOGLE_CLIENT_SECRET') GOOGLE_SECRET_KEY = config('GOOGLE_SECRET_KEY') GOOGLE_OAUTH_CALLBACK_URL = os.getenv('GOOGLE_OAUTH_CALLBACK_URL')

📦 Required Package: Install python-decouple if not already: pip install python-decouple

6

Restart Django Server

After adding the environment variables, restart your Django development server:

python manage.py runserver

🎉 Ready to Test!

Once you've completed all steps above, refresh this page to test Google Sign-In
